    Ingin deploy FastAPI ke server Ubuntu 24.04 dengan setup yang rapi, aman, dan mudah di-maintain? Panduan ini memandu kamu men-setup Gunicorn (ASGI server), Nginx (reverse proxy), dan HTTPS gratis dari Let‚Äôs Encrypt memakai Certbot. Kita juga pakai systemd supaya service otomatis jalan saat boot dan gampang di-restart kalau ada update.

    Ringkasan yang kamu dapat:

    • Struktur proyek sederhana FastAPI
    • Menjalankan app dengan Gunicorn (worker Uvicorn)
    • systemd service untuk start/stop/restart
    • Nginx reverse proxy ke Gunicorn
    • HTTPS otomatis (Certbot) + auto-renew
    • Firewall UFW (buka 80/443), log, dan tips troubleshooting

    Prasyarat

    • Server Ubuntu 24.04 (akses sudo)
    • Domain yang mengarah ke server (A/AAAA record)
    • Python 3.10+ (bawaan Ubuntu 24.04 sudah cukup)
    1. Siapkan struktur proyek di server

    Direktori yang rapi memudahkan otomatisasi.

    sudo mkdir -p /opt/fastapi/app
sudo adduser --system --group --home /opt/fastapi fastapi
sudo chown -R fastapi:fastapi /opt/fastapi
    1. Buat virtualenv dan install dependency
    sudo apt update
sudo apt install -y python3-venv

sudo -u fastapi python3 -m venv /opt/fastapi/venv
sudo -u fastapi /opt/fastapi/venv/bin/pip install --upgrade pip
sudo -u fastapi /opt/fastapi/venv/bin/pip install fastapi uvicorn gunicorn
    1. Buat aplikasi FastAPI minimal
    sudo -u fastapi tee /opt/fastapi/app/main.py >/dev/null <<'PY'
from fastapi import FastAPI

app = FastAPI()

@app.get("/healthz")
def healthz():
    return {"status": "ok"}

@app.get("/")
def root():
    return {"message": "Hello from FastAPI on Ubuntu 24.04!"}
PY

    Uji lokal (opsional):

    sudo -u fastapi /opt/fastapi/venv/bin/uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

    Kunjungi http://SERVER_IP:8000 untuk cek.

    1. Jalankan via Gunicorn (ASGI) secara manual (tes)
    sudo -u fastapi /opt/fastapi/venv/bin/gunicorn \
  -k uvicorn.workers.UvicornWorker \
  -w 2 \
  -b 127.0.0.1:8000 \
  app.main:app

    Kalau output terlihat normal dan port 8000 menerima request (coba curl 127.0.0.1:8000/healthz), lanjut ke pembuatan service.

    1. Buat systemd service untuk Gunicorn
    sudo tee /etc/systemd/system/fastapi.service >/dev/null <<'SERVICE'
[Unit]
Description=FastAPI app with Gunicorn
After=network.target

[Service]
User=fastapi
Group=fastapi
WorkingDirectory=/opt/fastapi
Environment="PATH=/opt/fastapi/venv/bin"
ExecStart=/opt/fastapi/venv/bin/gunicorn -k uvicorn.workers.UvicornWorker -w 2 -b 127.0.0.1:8000 app.main:app
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
SERVICE

sudo systemctl daemon-reload
sudo systemctl enable --now fastapi
sudo systemctl status fastapi --no-pager
    1. Pasang dan konfigurasi Nginx (reverse proxy)
    sudo apt install -y nginx

sudo tee /etc/nginx/sites-available/example.com >/dev/null <<'NGINX'
server {
    listen 80;
    listen [::]:80;
    server_name example.com www.example.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
    }
}
NGINX

sudo ln -s /etc/nginx/sites-available/example.com /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
    1. Buka firewall (UFW) untuk HTTP/HTTPS
    sudo ufw allow 'Nginx Full'   # membuka 80/tcp dan 443/tcp
sudo ufw status
    1. Pasang HTTPS gratis dengan Certbot
    sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d example.com -d www.example.com

    Certbot akan mengonfigurasi blok server 443 dan menambahkan auto‚Äërenew. Kamu bisa cek simulasi perpanjangan:

    sudo certbot renew --dry-run
    1. Cek dan monitoring
    • Coba: curl -I https://example.com/healthz
    • Log app: journalctl -u fastapi -f
    • Log Nginx: /var/log/nginx/access.log dan error.log
    1. Update aplikasi (rolling manual)
    sudo -u fastapi /opt/fastapi/venv/bin/pip install -U -r /opt/fastapi/requirements.txt || true
sudo systemctl restart fastapi

    Kalau kamu deploy dari Git:

    cd /opt/fastapi
sudo -u fastapi git pull
sudo systemctl restart fastapi

    Tips & Troubleshooting

    • 502 Bad Gateway: cek systemctl status fastapi dan journalctl -u fastapi -f.
    • Port konflik: pastikan hanya satu service yang bind ke 127.0.0.1:8000.
    • Timeouts: tingkatkan proxy_read_timeout atau optimalkan handler.
    • Keamanan: tambahkan security headers di Nginx, batasi body size jika perlu (client_max_body_size).
